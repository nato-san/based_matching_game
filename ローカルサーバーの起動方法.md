# ローカルサーバーの起動方法

## 🔍 問題

デスクトップの`index.html`を直接開いている場合（`file://`プロトコル）：
- MetaMaskが動作しない可能性があります
- Web3プロバイダーが正しく動作しない可能性があります

## ✅ 解決方法

ローカルサーバーを起動して、`http://localhost`で開く必要があります。

---

## 🚀 方法1: Pythonを使う（簡単）

### ステップ1: ターミナルを開く

1. `Command + スペース` → 「ターミナル」と入力
2. ターミナルを開く

### ステップ2: プロジェクトフォルダに移動

```bash
cd Desktop/based_matching_game
```

### ステップ3: サーバーを起動

```bash
python3 -m http.server 8000
```

または

```bash
python -m http.server 8000
```

### ステップ4: ブラウザで開く

1. ブラウザを開く
2. 以下のURLを入力：

```
http://localhost:8000
```

### ステップ5: サーバーを停止

ターミナルで `Ctrl + C` を押す

---

## 🚀 方法2: Node.jsを使う（npx serve）

### ステップ1: ターミナルを開く

### ステップ2: プロジェクトフォルダに移動

```bash
cd Desktop/based_matching_game
```

### ステップ3: サーバーを起動

```bash
npx serve
```

### ステップ4: ブラウザで開く

表示されたURL（例: `http://localhost:3000`）を開く

---

## 🚀 方法3: VS CodeのLive Server拡張機能を使う

### ステップ1: VS Codeを開く

### ステップ2: Live Server拡張機能をインストール

1. VS Codeで拡張機能を開く（`Command + Shift + X`）
2. 「Live Server」を検索
3. インストール

### ステップ3: サーバーを起動

1. `index.html`を開く
2. 右下の「Go Live」ボタンをクリック
3. ブラウザが自動的に開きます

---

## 🚀 方法4: デプロイされたURLを使う（最も簡単）

既にデプロイされている場合は、そのURLを使います：

```
https://based-matching-game-32xs.vercel.app/
```

このURLで開けば、MetaMaskが正しく動作します。

---

## 📊 各方法の比較

| 方法 | 難易度 | 必要なもの |
|------|--------|----------|
| Python | ⭐ 簡単 | Python（Macには標準で入っている） |
| npx serve | ⭐⭐ 普通 | Node.js |
| Live Server | ⭐⭐ 普通 | VS Code |
| デプロイされたURL | ⭐ 最も簡単 | なし（既にデプロイ済み） |

## 🎯 推奨方法

**最も簡単な方法：**
デプロイされたURLを使う：
```
https://based-matching-game-32xs.vercel.app/
```

**ローカルで試したい場合：**
Pythonを使う方法が最も簡単です。

---

## ⚠️ 注意事項

- `file://`プロトコルではMetaMaskが動作しない可能性があります
- 必ず`http://localhost`または`https://`で開いてください
- サーバーを起動したままにしておく必要があります

---

## 💡 ヒント

### サーバーが起動しているか確認

ブラウザのアドレスバーで確認：
- ✅ `http://localhost:8000` → 正しい
- ❌ `file:///Users/...` → 間違い（サーバーを起動する必要がある）
